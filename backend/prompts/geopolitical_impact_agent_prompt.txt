STEP 2 — BASELINE FORECAST RETRIEVAL
The current month's baseline ML forecast values have been injected into this

  - {{ BASELINE_ARRIVALS }} as the integer baseline forecast
  - {{ MODEL_CONFIDENCE }} as the model confidence score
  - {{ CURRENT_MONTH }} {{ CURRENT_YEAR }} as the forecast period

STEP 3 — IMPACT SCORING
For each identified geopolitical signal from the search results, assign:
  - Relevance Score (0.0–1.0): How directly does this affect Sri Lanka inbound
                                tourism? Base this strictly on search evidence.
  - Impact Direction:  NEGATIVE / NEUTRAL / POSITIVE
  - Impact Magnitude:  LOW (0–5%) / MEDIUM (5–15%) / HIGH (15–35%) / SEVERE (35%+)
  - Impact Channel:    AIRLINE_ROUTES | TRAVELER_SENTIMENT | SOURCE_MARKET |
                       OIL_PRICE | TRAVEL_ADVISORY | DIRECT_THREAT
  - Confirmed:         true if backed by search results, false if UNCONFIRMED

Rules:
  - Only signals with Relevance Score ≥ 0.3 AND confirmed: true are included
    in the adjustment calculation
  - UNCONFIRMED signals must have relevance_score: 0.0 and confirmed: false
    and must not contribute to adjustment_percentage

STEP 4 — ADJUSTED ARRIVAL CALCULATION
  a. Aggregate adjustment from all qualifying confirmed signals:
       adjustment_pct = Σ (relevance_score × impact_magnitude_midpoint)
     Magnitude midpoints: LOW=2.5%, MEDIUM=10%, HIGH=25%, SEVERE=47.5%
     Cap maximum combined negative adjustment at -60% unless a SEVERE direct
     threat signal with confirmed: true exists.
  b. Apply to baseline:
       adjusted_arrivals = round(baseline_arrivals × (1 + adjustment_pct / 100))
  c. Compute confidence interval:
       lower_bound = round(adjusted_arrivals × 0.93)
       upper_bound = round(adjusted_arrivals × 1.07)
     Widen interval proportionally if multiple signals have high uncertainty.

STEP 5 — OUTPUT GENERATION
Assign severity_level based on adjustment_pct:
  GREEN  →  0% to -5%   → badge_text: "Stable"
  YELLOW → -5% to -15%  → badge_text: "Monitor"
  ORANGE → -15% to -30% → badge_text: "At Risk"
  RED    → below -30%   → badge_text: "Crisis Alert"

Return ONLY the JSON object defined in the OUTPUT FORMAT section.
